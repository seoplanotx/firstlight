# Onboarding flow

## Step 1 — Welcome
- explains what OncoWatch is
- explains what it is not
- shows the safety disclaimer clearly

## Step 2 — Patient Profile
- profile name
- optional display name / initials
- optional DOB
- cancer type
- subtype
- stage / context
- biomarkers
- therapy history
- location / travel radius
- preferences and exclusions
- notes

## Step 3 — OpenRouter
- plain-language explanation
- open OpenRouter button
- “How to get your API key” panel
- API key field
- model selector
- test key action
- save provider settings

## Step 4 — Monitoring Preferences
- daily run time
- report defaults
- source toggles
- demo profile option

## Step 5 — Test Run
- local storage
- database
- source health
- report generation
- OpenRouter validation if configured

## Step 6 — Completion
- confirms setup
- shows next daily run
- explains that reports are stored locally
- opens the dashboard
